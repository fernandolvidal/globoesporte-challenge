DESCRIÇÃO DE PASTAS:


1. ORIGINAL DATA
Nessa pasta se encontram os arquivos originais de treino. Essa pasta padrão é utilizada para guardar os dados originais de maneira que os mesmos fiquem intocados durante todo o projeto. Ela será entregue vazia, caso necessite testar, é necessário colocar os arquivos originais aqui.


2. PROCESSED DATA
Pasta que guardará todos os dados intermediários, semi-processados do processo de análise. Será entregue vazia, os arquivos intermediários são produzidos executando os scripts.


3. UPLOADED DATA
Pasta não utilizada para este projeto.


4. ANALYSIS
Aqui estão guardados os scripts utilizados no processo de análise dos dados originais. Tais scripts são responsáveis por ler os dados originais, processá-los e por final salvar na pasta 2. PROCESSED DATA.


5. INSIGHTS
Pasta que guarda arquivos informativos de todo o processo de leitura, pre-processamento e teste. Aqui ficam as conclusões e ideias desenvolvidas durante o projeto.


6. FINAL
Script que representa o modelo final escolhido. Organizado de forma que o mesmo execute ponta-a-ponta.
